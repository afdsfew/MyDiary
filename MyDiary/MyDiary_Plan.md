# 기말 프로젝트 기획서 – 마이 다이어리 (My Diary)

## 1. 프로젝트 개요

**프로젝트 설명**  
마이 다이어리는 하루 동안 해야 할 일(To-Do)과 짧은 메모(다이어리)를 한 화면에서 함께 관리하는 iOS 앱이다.  
사용자는 할 일을 추가·삭제·완료 체크할 수 있으며, 하루가 끝날 때 간단한 메모를 기록하여 하루의 흐름을 정리할 수 있다.  
SwiftUI + CoreData 기반으로 구현하며, 날짜별로 할 일과 메모를 묶어 관리하는 ‘하루 단위 기록 앱’이다.

**개발 동기**  
기존 할 일 앱은 체크 관리는 편리하지만 감정이나 상황 기록이 어렵고, 일기 앱은 작업 관리 기능이 부족하다.  
따라서 할 일과 메모를 결합해 하루를 통합적으로 관리할 수 있는 앱을 제작하고자 한다.

---

## 2. 레퍼런스 앱 (벤치마킹)

### To Do List – 미리 할 일 / 투두리스트

**참고 기능**
- 할 일 추가(Add Task)
- 완료 처리(Check/Uncheck)
- 카테고리 분류(Category Tag)
- 오늘 할 일 중심 정렬(Today View)

**참고 UI 요소**
- 좌측 체크박스 + 우측 제목 구조
- 리스트(List) 기반 화면
- 항목 우측 카테고리/마감일 작은 표시
- 상단 우측 ‘+’ 버튼 배치

---

### 플래닛(PLANIT)

**참고 기능**
- 날짜(dayKey) 단위 데이터 묶음
- 하루 기록 리스트(Daily Summary)
- 메모/일기 작성 기능
- 날짜 선택 후 상세 화면 이동

**참고 UI 요소**
- 상단 날짜 헤더(Date Header)
- 하루 카드뷰(Card UI)
- 하단 메모 입력(TextEditor)
- 달력/리스트 기반 날짜 탐색 구조

---

## 3. 구현 기능

**핵심 기능**
- 할 일 추가/수정/삭제/완료 체크
- 카테고리 선택(공부/개인/과제 등)
- 오늘 메모 작성 및 저장(TextEditor)
- CoreData 저장(Todo/Diary 엔티티)
- 날짜별 기록 조회(완료 개수 + 상세 화면)

**확장 기능(선택)**
- 검색 기능(제목/메모)
- 주간 완료 통계(텍스트 기반)

---

## 4. AI 협업 전략

**AI가 담당할 작업**
- 유사 앱 기능/구조 조사
- 화면 구조 및 기능 초안 제안
- SwiftUI + CoreData 코드 초안 생성
- 에러 원인 분석 및 리팩토링 아이디어 제시
- 문서 정리 보조(PROJECT_OVERVIEW.md 등)

**사용자(본인)가 담당할 작업**
- 전체 구조 및 기능 설계 확정
- CoreData 엔티티 설계 및 데이터 흐름 결정
- ViewModel 핵심 로직 구현
- 시뮬레이터 테스트 및 디버깅
- 최종 UI/UX 조정과 완성도 관리

**참고 문서 관리 방식 (Gemini.md, Claude.md, Agents.md 기반)**
- 사람용 문서: `README.md`
- AI 협업용 문서: `AI_RULES.md`, `FEATURE_SPEC.md`, `DATA_MODEL.md`
- 목적: 기능 정의, 데이터 구조, 화면 흐름을 명확히 기록하여 AI가 일관된 규칙 안에서 코드 생성/수정 가능하도록 함

---

## 정리

마이 다이어리는 "할 일 관리 + 하루 메모"를 결합한 통합형 앱으로,  
SwiftUI와 CoreData 학습 목적에 적합하며,  
AI 협업 방식을 문서 기반으로 적용할 수 있는 프로젝트이다.
